burpsuite截获流量

  • burpsuite
tools 
burpsuite的测试功能，主要体现在能灵活的修改重放http请求，其原理是将burpsuite做为代理，在客户端和服务器之间截断请求．默认不截获返回内容．（可以分析下httpproxy.pcapng，了解下http代理．）

https证书问题

web请求有http和https两种，https是经过ssl证书加密的http通信，burpsuite在中间做代理，则必须让客户端信任burp的证书．
浏览器设置代理指向burp监听的端口，然后访问 https://burp 把证书导入就行了

客户端支持代理的解决方案

直接设置．

可见代理，适用于支持代理的客户端

客户端不支持http代理的解决方案

dns欺骗

通过修改客户端的hosts文或者其他办法把发向待测试的web应用程序的流量引到burpsuite上．这就需要burpsuite来判断如何转发流量．当在被测试的客户端与burp在同一机器时，为使其能正常访问，可设置burp内部的域名解析，来覆盖系统hosts文件的配置．
Hostname Resolution配置

不可见代理（透明代理）

代理服务器读取数据包的URL和host头，来判断客户端要访问的网站．
配置位置：
Proxy -> Option -> Proxy Listeners
点击 Edit 某个代理的配置，切换到 Request handing 标签页，勾选 Support invisible proxying，不能随便勾选，如果勾选了此项则不支持可见代理了．

请求没有host头

这种情况单靠不可见代理代理，也不能解决问题了．
这时候需要让burp自己来确定如何转发．在Request handing 标签页下来配置．
当客户端访问多个域名，多个待测服务器时，可以启动多个监听，运用linux子接口机制，让测试机有多个ip地址，来完成测试．

总结

苑老师的表格

linux lo子接口配置（kali）

ifconfig lo:1 127.0.0.2 netmask 255.255.255.0
ifconfig lo:1 up

这样就创建好了
在burp添加代理的时候，就能看见这个地址了，然后在浏览器里直接配置代理就可以用了．

使用完毕，把接口关闭就恢复原样了．

ifconfig lo:1 down
